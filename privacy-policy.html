Privacy Policy

Application: Automatizaciones n8n · Last updated: 2026-01-14

Automatizaciones n8n is a personal automation application used to run workflows locally using n8n (self-hosted in a Docker container).

This application connects to Google services (Gmail, Google Calendar, Google Drive and Google Analytics) only to execute user-configured automations.

Data controller

This application is for personal use only. The owner/operator of the application is the developer of the Google Cloud OAuth client used to access Google APIs.

Data processed

Depending on the permissions granted, this application may access and process the following data:

Gmail

  • Read and search email messages (content and metadata)
  • Move messages between labels/folders
  • Create and modify labels
  • Create drafts
  • Send emails (e.g., automated notifications)

Google Calendar

  • Create events
  • Read/list events
  • Modify/update events
  • Delete events (when required by the workflow)

Google Drive (including Google Docs/Sheets)

  • Create, rename, move and delete folders
  • Upload, rename, move and delete files
  • Create, modify and delete Google Workspace documents (Docs, Sheets, etc.)

Google Analytics

  • Read and download analytics data for reporting/analysis
  • No configuration or administrative changes are performed unless explicitly configured

Purpose of processing

Data is processed exclusively for personal automation purposes, such as:

  • Email organization (labelling, moving, filtering)
  • Automated notifications and message sending
  • Calendar scheduling automation
  • Document generation and management in Drive (Docs/Sheets)
  • Analytics reporting and personal analysis

Data storage

This application does not permanently store Gmail messages, calendar events, Drive files or Analytics data. Data is processed locally for automation purposes and then discarded.

Some transient data may appear in execution logs within n8n (for example, workflow execution metadata). The user can configure n8n to minimize or disable saving execution data.

Credentials and tokens

OAuth tokens required to access Google APIs are stored locally by the self-hosted n8n instance running inside a Docker container. n8n stores credentials using its own credential encryption/storage mechanisms.

Data sharing

No data is sold or shared with third parties. All processing happens locally on the user's environment unless a workflow explicitly sends data to an external service chosen by the user.

Security

The n8n instance is private and protected by authentication. Access is restricted to the user's local/private environment.

User controls and revocation

You can revoke this application's access at any time from your Google Account settings: Security → Third-party access (or “Manage third-party access”). After revocation, the application will no longer be able to access Google services.
